## 3. Directory Structure on Server

```
/var/www/diar/
├── app/                         ← Git repo clone (Next.js application)
│   ├── .env                     ← Production secrets (NOT in Git)
│   ├── .next/                   ← Build output (generated by npm run build)
│   ├── node_modules/
│   ├── prisma/
│   │   └── migrations/
│   ├── public/
│   │   └── uploads → /var/www/diar/uploads/   ← SYMLINK (critical)
│   ├── scripts/
│   │   └── deploy.sh
│   └── ecosystem.config.cjs
│
└── uploads/                     ← Persistent storage (survives deploys)
    ├── properties/              ← Villa gallery images (ganje-*.png)
    └── receipts/                ← Employee payment receipt images
```

> **The symlink at `public/uploads` is load-bearing.** If it becomes a real directory instead of a symlink, uploads survive but the deploy.sh script would not know to preserve them. Always verify: `ls -la /var/www/diar/app/public/uploads` should show `uploads -> /var/www/diar/uploads`.

## 4. PM2 Configuration

Defined in `ecosystem.config.cjs`:

```javascript
{
  name: "diar",
  cwd: __dirname,
  script: "node_modules/next/dist/bin/next",
  args: "start -p 3000",
  instances: 1,
  exec_mode: "fork",
  autorestart: true,
  max_memory_restart: "512M",
  env: { NODE_ENV: "production", PORT: 3000 }
}
```

**Useful PM2 commands (run on server only, SSH):**
```bash
pm2 status diar          # Check running status
pm2 logs diar            # View real-time logs
pm2 logs diar --lines 100  # Last 100 log lines
pm2 restart diar         # Restart (used in deploy.sh)
pm2 save                 # Persist current process list
```

## 5. Nginx Configuration

The template is in `nginx.diar.conf.example`. Active config is at `/etc/nginx/sites-available/diar` on the server.

Key rules:
- HTTP (port 80) reverse proxies to `localhost:3000`
- `/uploads/` path is served directly from `/var/www/diar/uploads/` (bypasses Node.js)
- HTTPS (port 443) block is commented out — SSL not yet configured

Required proxy headers (necessary for cookie `secure` detection):
```nginx
proxy_set_header X-Forwarded-Proto $scheme;
proxy_set_header Host $host;
```

**Current state:** HTTP only. HTTPS pending domain connection.

## 8. Deployment Flow

### Automatic Deploy (target state)
```
Developer → git push origin main
              ↓
         GitHub Actions
              ↓
         SSH to Droplet
              ↓
         bash scripts/deploy.sh
              ↓
         Site updated
```

### What `deploy.sh` Does (in order)
1. Preflight: checks `package.json`, `.env`, `DATABASE_URL` presence
2. Warns if `public/uploads` is missing (non-fatal)
3. `git pull` (updates code)
4. `npm ci` (clean install)
5. `npx prisma generate` (regenerate Prisma client)
6. `npx prisma migrate deploy` (apply pending migrations — safe, idempotent)
7. `npm run build` (Next.js production build)
8. `pm2 restart diar` (or `pm2 start` if first run)
9. `pm2 save`

**deploy.sh NEVER runs:** `prisma seed`, `prisma migrate reset`, `prisma db push`, upload deletion, database wipes.

### Manual Deploy (SSH fallback)
```bash
ssh deploy@165.245.245.149
cd /var/www/diar/app
bash scripts/deploy.sh
```

## 10. SSL / Domain Status

| Status | Detail |
|--------|--------|
| Domain | **Not connected yet** |
| SSL certificate | **Not installed** |
| Certbot | Installed on server (via server-setup.sh) |
| HTTPS nginx block | Commented out in config template |

**To enable SSL:**
1. Point your domain's A record to `165.245.245.149`
2. Update `server_name` in `/etc/nginx/sites-available/diar`
3. Run: `sudo certbot --nginx -d yourdomain.com`
4. Reload nginx: `sudo systemctl reload nginx`
5. Test HTTPS login

After SSL is enabled, the login cookie will switch to `secure: true` + `sameSite: "none"` automatically (the route detects `https:` in the request URL).

## 11. Backup Strategy

Script: `scripts/backup.sh`

Creates at `backups/YYYY-MM-DD-HH-mm/`:
- `database.dump` — PostgreSQL custom-format dump
- `uploads.tar.gz` — all files in `/var/www/diar/uploads/`

**Current status:** No scheduled backup cron. Must be set up.

Recommended cron (daily at 3am):
```bash
crontab -e
# Add:
0 3 * * * cd /var/www/diar/app && bash scripts/backup.sh >> /var/log/diar-backup.log 2>&1
```

**Restore database (WARNING: destructive):**
```bash
pg_restore -d property_agah --clean --if-exists backups/YYYY-MM-DD-HH-mm/database.dump
```
